# UX/UI Audit Template

Use this audit to identify what is not working and how to make the UI feel professional.

## 1) Context & Goals
- Product goal and primary KPI
- Target users (experience level, device, context)
- Primary task (one per screen)
- Constraints (brand, tech, legal)

## 2) User Journey Review
- Steps from entry to success
- Drop-off points and hesitation moments
- Missing information or reassurance
- Critical moments that need feedback

## 3) Information Architecture
- Is content grouped logically?
- Are labels and terminology consistent?
- Can users find the primary action in 3 seconds?

## 4) Visual Hierarchy
- One clear primary focus per screen
- Secondary actions visibly subordinate
- Title, subtitle, body: clear scale
- Spacing and alignment: consistent rhythm

## 5) Interaction & Feedback
- Clear affordances for clickable elements
- Immediate feedback for actions
- Loading, empty, error states defined
- Focus states visible for keyboard

## 6) Content & Microcopy
- Short, concrete copy
- Reduce cognitive load
- Explain next step plainly
- Error messages: helpful and specific

## 7) Accessibility Basics
- Contrast meets minimum standards
- Font sizes readable on mobile
- Touch targets >= 44px
- Forms with labels and focus

## 8) Trust & Credibility
- Social proof or reassurance where needed
- Consistent tone and branding
- Avoid visual noise and clutter

## 9) Visual Polish Checklist
- Consistent border radius and shadows
- Icon style consistency
- Image quality and alignment
- Avoid unnecessary gradients or effects

## 10) Priority & Recommendations
- List issues by impact (High, Medium, Low)
- Suggest fixes and rationale
- Provide quick wins vs longer-term changes

## Output Format
- Brief summary of main UX risks
- Bullet list of issues with file/section references
- Proposed fixes and expected impact
