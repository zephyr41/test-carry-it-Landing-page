# Diagnostic

## Overview

Analyser un problème front (HTML/CSS/JS), expliquer ce qui ne fonctionne pas, et proposer un diagnostic clair sans appliquer de changements.

## Workflow

1. Identifier la demande et l'impact attendu (rendu, interaction, navigation, console).
2. Lire les fichiers pertinents et établir le contexte minimal (HTML, CSS, JS).
3. Reproduire mentalement le flux : DOM ciblé, styles appliqués, événements JS, dépendances.
4. Lister les causes probables, avec preuves dans le code.
5. Proposer un plan de correction et demander validation si une modification est nécessaire.

## Output attendu

- Donner un diagnostic précis, avec références de fichiers et fonctions.
- Expliquer le “pourquoi” (ex: sélecteur cassé, script chargé trop tôt, CSS plus spécifique).
- Proposer des solutions, sans modifier de fichiers.
- Poser des questions ciblées si une info manque (erreurs console, étapes exactes).
