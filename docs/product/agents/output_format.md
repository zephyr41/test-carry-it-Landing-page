 Output Format — CarryIT Skill

## 1. Reformulation
Résume le besoin en 2 à 4 lignes maximum.

## 2. Ce qui manque
Liste uniquement les informations manquantes qui changent réellement la structure UX/UI.
Maximum : 5 questions.

## 3. Proposition d’interface
Décris l’écran ou le flow proposé :
- objectif de l’interface
- action principale
- blocs à afficher
- ordre des blocs
- CTA principal
- informations secondaires

## 4. Logique UX
Explique brièvement pourquoi cette structure est cohérente avec :
- le besoin utilisateur
- la vérité produit
- la clarté d’usage

## 5. Points de vigilance
Liste les risques principaux :
- confusion
- surcharge
- friction
- hypothèses critiques

## 6. Prompt prêt à réutiliser dans Claude
Fournis un bloc court que l’utilisateur peut redonner tel quel à Claude pour approfondir, corriger ou générer une variante.
